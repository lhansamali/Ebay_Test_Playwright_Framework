## 🎯 Project Overview

This framework automates testing of eBay's related products functionality, including:
- Validating that exactly 6 related products are displayed
- Verifying all related products have "Top Rated Plus" status
- Ensuring related product prices are within ±$5 of the main product
- Confirming related products belong to the same category as the main product
- Testing that related product details open in new tabs

## 📁 Folder Structure

```
Ebay_Test_Playwright_Framework/
│
├── config.ts                      # Additional Playwright configuration
├── package.json                   # Project dependencies and scripts

│
├── fixtures/                      # Test fixtures and base setup

│
├── test/                          # Test specifications
│   └── related-products.spec.ts  # Related products test cases
│

│
└── Utils/                         # Utility classes and helper functions
```

### 📁 Folder Purposes

- **`fixtures/`**: Contains base test fixtures that handle browser setup, anti-bot configurations, and common test setup procedures
- **`pages/`**: Page Object Model classes that encapsulate web page interactions and element selectors
- **`reports/`**: Stores HTML test reports and execution summaries generated by Playwright
- **`test/`**: Contains all test specification files organized by feature/functionality
- **`test-results/`**: Automatic storage for test artifacts like screenshots, videos, and execution traces
- **`Utils/`**: Helper classes and utility functions for data processing and common operations


### 2. Install Dependencies

```bash

```

### 3. Install Playwright Browsers

```bash
npx playwright install
```

```bash

```

## 🛡️ Anti-Bot Detection Setup

### Why playwright-extra and playwright-extra-plugin-stealth?

eBay, like many e-commerce platforms, employs sophisticated bot detection mechanisms to prevent automated scraping and testing. The standard Playwright browser instances can be easily detected and blocked. Here's why these additional packages are essential:

#### **playwright-extra**
- Provides a plugin system for Playwright
- Allows integration of stealth plugins and other extensions
- Enables advanced browser manipulation beyond standard Playwright capabilities

#### **playwright-extra-plugin-stealth**
- **Removes WebDriver Flags**: Eliminates `navigator.webdriver` property that identifies automated browsers
- **Spoofs Chrome Runtime**: Makes the browser appear as a regular Chrome instance
- **Handles Automation Indicators**: Removes various automation-related properties and behaviors
- **Permission Spoofing**: Manages permission-related detection vectors
- **Plugin Fingerprinting**: Reduces automated browser fingerprinting detection

### Implementation in This Framework

The anti-bot configuration includes:

1. **Browser Launch Options**:
   ```typescript
   args: ['--disable-blink-features=AutomationControlled']
   ```

2. **WebDriver Property Removal**:
   ```typescript
   await page.addInitScript(() => {
       Object.defineProperty(navigator, 'webdriver', {
           get: () => undefined,
       });
   });
   ```

3. **Realistic User Agent**: Mimics actual Chrome browser user agents
4. **Proper Headers**: Includes Accept-Language and other realistic headers
5. **Headless Disabled**: eBay detection is more effective against headless browsers

### When These Packages Are Required

- **E-commerce Testing**: Platforms like eBay, Amazon, etc.
- **Anti-Bot Protection**: Sites with Cloudflare, bot detection services
- **Social Media Platforms**: Instagram, Facebook, LinkedIn testing
- **Financial Services**: Banking and fintech applications
- **High-Security Sites**: Government or enterprise applications

Without these packages, you might encounter:
- Immediate redirects to CAPTCHA pages
- Access denied errors (403/429 status codes)
- Suspicious activity warnings
- Rate limiting or IP blocking
- Incomplete page loads or missing content

## 🔧 Troubleshooting

**Common Issues:**

1. **Tests Failing Due to Bot Detection**:
   - Ensure Chrome is installed and used as the browser
   - Verify stealth plugins are properly installed
   - Check that headless mode is disabled

2. **Slow Test Execution**:
   - This is normal due to anti-detection measures
   - Consider running tests in parallel cautiously

3. **Element Not Found Errors**:
   - eBay may update their DOM structure
   - Update selectors in [SearchPage.ts](pages/SearchPage.ts)
   - Verify page load timing configuration
